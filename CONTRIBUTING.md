# Contributing

## How to Contribute

1. Fork the repository and create a branch from `main`.
2. Keep commits clear and scoped to one concern.
3. Run local simulation checks before opening a pull request.
4. Include reasoning, assumptions, and validation notes.

## Coding Guidelines

- Keep MATLAB functions small and focused.
- Prefer explicit variable names over abbreviations.
- Validate input assumptions where practical.
- Avoid changing unrelated code or formatting.

## Pull Request Expectations

- Problem statement
- Summary of technical change
- Validation evidence (plots, metrics, or command output)
- Documentation updates if behavior changed

## Reporting Issues

Use issue templates to report bugs or propose enhancements. Include reproducible steps and environment details.
